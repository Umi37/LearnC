## 나의 풀이

1번 정답
B, D

2번 정답
rand()는 호출 순서가 동일하고 사용환경이 동일하고, 시드가 동일하다면 동일한 결과를 낸다. 따라서 진짜 난수라고 볼수는 없다. 프로그램 A와 B에서 rand() 함수는 모두 2번 호출되었고 시드는 1로 설정되었으나, 콘솔에 출력하는 결과를 A는 1번째와 2번째 호출 결과값까지 출력하였으나, B는 두번째 호출의 결과값만 출력하였으므로
A의 콘솔 출력값중 첫번째 rand() 함수의 출력값이 제외된 결과만 출력되게 된다.
2번째 rand() 호출로 인한 출력값은 양쪽 프로그램의 실행결과로 모두 콘솔에 출력되는데, 이는 rand() 함수가 내부에 동일한 연산으로 실행되기 때문이며, 동일한 시드를 출발점으로 삼았기 때문이다. rand() 함수 내에서 내부동작 방식을 살펴보면, rand() 함수는 전역함수처럼 동작하며 앞선 연산결과를 가리키는 포인터를 인자로 받아 연산에 활용하므로, 시드 즉 출발점이 동일하다면 항상 동일한 시퀀스를 가진다는 특징이 있다. 
따라서 A프로그램과 B프로그램의 실행결과는 B프로그램의 결과가 A프로그램의 결과에 마치 교집합처럼 포함되는 형태를 띄게된다.

문제 3. D

문제 4. 헉 모름

문제 5. 문제 5번의 정답은 문제 2번에서 설명한 답과 일맥상통하다. 문제 2번의 답에 덧붙혀 해결방안을 모색해본다면, 시드로 time(NULL) 을 이용하는 방법과 사용자정의함수를 통해 생성된 시퀀스의 수열을 뒤틀어버릴 수 있다. 이 사용자 정의 함수에도 rand() 함수를 사용하여 몇번째 수열을 출력하지 않을것인지에 대해서도 난수값을 전달할수도 있겠으나, rand() 함수는 전역함수로써, 앞선 실행결과를 모두 기억하고 있으므로 다중스레드에서 함께 사용시, 다른 스레드의 결과에 영향을 미칠수 있으므로 다른방법을 모색하는 것이 좋을 듯하다.
