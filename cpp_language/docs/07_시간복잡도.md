## 📚 시간복잡도(Big-O) 기본 정리

### 📌 시간복잡도란?
- "데이터 양이 커질 때, 알고리즘이나 동작이 얼마나 빨라지거나 느려지는지를 나타내는 척도"를 의미함.

### 📌 주요 시간복잡도 예시

| 표기 | 의미 |
|:---|:---|
| O(1) | 데이터 양과 관계없이 "항상 같은 시간" 안에 끝남 (엄청 빠름) |
| O(N) | 데이터 개수(N)에 비례해서 시간이 걸림 |
| O(N²) | 데이터 개수의 제곱(N²)만큼 시간이 걸림 (ex: 이중 for문) |

## 📚 벡터(vector)에서의 시간복잡도

### 📌 push_back 동작

| 상황 | 설명 | 시간복잡도 |
|:---|:---|:---|
| 여유 공간이 있을 때 | 바로 메모리에 추가 | O(1) |
| 공간이 부족할 때 | 새 배열을 만들고, 기존 데이터를 모두 복사 | O(N) |

---

### 📑 동작 흐름 비유

- 여유 공간이 있을 때 (O(1))
  ```
  [10][20][ ][ ][ ][ ]
  → push_back(30)
  → 바로 [10][20][30][ ][ ][ ]
  ```

- 공간이 부족할 때 (O(N))
  ```
  [10][20][30] (공간 없음)

  새 배열 생성 → [10][20][30][ ][ ][ ][ ][ ][ ]
  데이터 복사 (N개 복사)

  push_back(40) → [10][20][30][40][ ][ ][ ][ ][ ][ ]
  ```

---

### ✨ 한 줄 요약

> **벡터의 push_back은 대부분 O(1)로 빠르지만, 메모리가 부족해 확장할 때는 O(N) 복사 비용이 발생할 수 있다.**
